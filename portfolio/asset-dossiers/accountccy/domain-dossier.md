The CCY State Chain — Core Doctrine

This is the original intellectual property of the accountccy.com asset. It is not derived from any single existing framework. It synthesizes the currency behavior described in Swift ISO 20022 messaging standards, SAP and Oracle ERP account configuration logic, IFRS functional currency and translation doctrine, and CFO-level FX risk frameworks into a single unified state model.

The eight states of the CCY State Chain are:

State 01 — Quoted Currency
Currency as a price in a market. No accounting consequence has occurred. This is rate, not record.

State 02 — Transaction Currency
The currency in which a financial event is denominated. The rate is locked. The accounting obligation is born.

State 03 — Settlement Currency
The currency in which payment is actually made. May differ from transaction currency. Divergence creates FX consequences.

State 04 — Account Currency
The critical inflection point. The currency assigned to the account in which the transaction is recorded. This is where external movement becomes internal accounting state. This is the domain of accountccy.com.

State 05 — Ledger Currency
The functional currency of the legal entity. The currency in which the general ledger is maintained. Determined by primary economic environment, not management preference.

State 06 — Reporting Currency
The currency in which financial results are presented for management, regulatory, or investor purposes. Translation gains and losses emerge at this layer.

State 07 — Consolidation Currency
The group-level currency into which multiple entity ledgers are consolidated. Maximum complexity. Maximum FX translation exposure.

State 08 — Audit Reality
The terminal state. Currency figures that have passed audit, compliance review, board scrutiny, and regulatory examination. The defensible accounting truth.

Doctrine Rule

Account currency is not merely State 04 in isolation. It is the gate through which every subsequent state is determined. A misconfigured or under-governed account currency does not merely affect one account. It propagates distortion through ledger, reporting, consolidation, and audit layers. This is why account currency is a control discipline, not a configuration detail.
